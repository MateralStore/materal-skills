# MMB 功能设计

本技能提供 Materal.MergeBlock(MMB) 框架项目的功能设计流程。

## 核心原则

- **主从 Agent 协作模式**：主 Agent 负责整体协调，Sub Agent 并行处理各功能模块
- **模块化设计**：功能模块应职责单一、高内聚低耦合
- **可并行开发**：各功能模块尽量独立，支持并行开发
- **可验证性**：每个功能设计应包含明确的验收标准

## 前置条件

功能设计应在**需求分析完成后**进行。确保已完成：
- 功能概述收集
- 业务场景收集
- 实体信息收集
- API 接口收集
- 业务规则收集

## 工作流程

功能设计按以下三步执行：

### 步骤 1：主 Agent 初步拆分

基于需求分析结果，将功能拆分为多个独立的功能模块。

#### 拆分原则

1. **职责单一**：每个功能模块只负责一个明确的业务职责
2. **高内聚低耦合**：模块内部功能紧密相关，模块间依赖最小化
3. **符合 MMB 模块结构**：确定功能属于 Core、Main 还是其他业务模块
4. **可独立验证**：每个功能模块应有明确的验收标准

#### 拆分方法

分析需求文档中的：
- 业务实体 → 识别每个实体的 CRUD 操作
- API 接口 → 按业务领域分组接口
- 业务规则 → 识别独立的业务规则处理

#### 输出格式

```markdown

> 基于需求分析文档：[RequirementsAnalysis/{模块名}.md](../RequirementsAnalysis/{模块名}.md)
>
> 设计日期：{日期}
> 所属模块：{模块名}

---

## 功能模块拆分

### 功能模块 1：{功能名称}
- **功能编号**：F-{模块缩写}-01
- **简要描述**：{职责说明}

### 功能模块 2：{功能名称}
- **功能编号**：F-{模块缩写}-02
- **简要描述**：{职责说明}

...
```

### 步骤 2：Sub Agent 详细设计

为每个功能模块启动独立的 Sub Agent，并行进行详细设计。

#### Sub Agent 任务清单

每个 Sub Agent 需要完成以下设计内容：

1. **功能编号**：F-{模块缩写}-{序号}
2. **功能名称**：简洁明确的功能标识
3. **所属模块**：确定属于 Core、Main 还是其他模块
4. **职责描述**：该功能的具体职责和边界

### 设计详情

1. **输入参数**：参数名、类型、必填、验证规则（表格格式）
2. **控制器接口返回**：完整的 C# 方法签名（含 XML 注释）
3. **DTO 结构**：所有相关 DTO 的完整定义（含 XML 注释）
4. **处理逻辑**：编号步骤列表
5. **异常处理**：场景、异常类型、错误消息（表格格式）
6. **验收标准**：checkbox 格式的可验证条件
