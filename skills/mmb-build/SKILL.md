# MMB 项目构建

## 概述

此技能用于构建 MMB (Materal.MergeBlock) 框架项目的模块解决方案。

## 快速开始

### 基本用法

构建指定模块：
```bash
dotnet build <解决方案文件路径>
```

### 解决方案文件查找

MMB 模块通常使用 `.slnx` 或 `.sln` 格式的解决方案文件，位于模块根目录：

**命名模式：**
- `{ProjectName}.{ModuleName}.slnx`（优先）
- `{ProjectName}.{ModuleName}.sln`

**示例：**
- `ZhiTu.Main\ZhiTu.Main.slnx`
- `ZhiTu.Core\ZhiTu.Core.slnx`

### 查找解决方案文件

使用 Glob 工具搜索：
```bash
**/*.slnx
```
或
```bash
**/*.sln
```

## 工作流程

1. **确定模块** - 根据用户指定或当前目录确定要构建的模块
2. **查找解决方案文件** - 在模块目录中查找 `.slnx` 或 `.sln` 文件
3. **执行构建** - 运行 `dotnet build` 命令
4. **检查结果** - 验证构建是否成功，报告错误或警告

## 模块识别策略

当用户提及模块名称时，按以下优先级确定路径：

1. **用户明确提供的路径** - 直接使用用户指定的解决方案文件或目录
2. **当前工作目录** - 如果在模块目录内，查找当前目录的解决方案文件
3. **按名称搜索** - 使用 Glob 搜索匹配的解决方案文件
   - 搜索模式：`**/*<模块名>*.slnx` 或 `**/*<模块名>*.sln`
   - 示例：用户说 "构建 Main 模块" → 搜索 `**/*Main*.slnx`

### 示例场景

| 用户输入 | 查找模式 |
|---------|---------|
| "构建 Main 模块" | `**/*Main*.slnx` |
| "构建 Core" | `**/*Core*.slnx` |
| "构建当前模块" | 使用当前目录，查找 `*.slnx` |
| "构建 ZhiTu.Main" | `**/*ZhiTu.Main*.slnx` |

## 构建命令

```bash
# 指定解决方案文件构建
dotnet build <解决方案文件路径>

# 指定配置（默认 Debug）
dotnet build <解决方案文件路径> --configuration Release

# 不显示启动徽标
dotnet build <解决方案文件路径> -nologo
```

## 使用触发条件

在以下情况使用此技能：
- 代码生成完成后需要验证
- 用户明确要求"构建"、"编译"项目
- 修改代码后需要确认编译是否成功
- 开发流程中的构建验证阶段

## 注意事项

1. 优先查找 `.slnx` 文件，如果不存在则查找 `.sln` 文件
2. 一个模块目录通常只有一个解决方案文件
3. 如果找到多个匹配文件，优先选择与模块名最相关的
