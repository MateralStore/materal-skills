# MMB 代码生成器

## 概述

此技能自动化 MMB (Materal.MergeBlock) 框架项目的代码生成。在修改实体、配置或构建项目之前都需要运行代码生成。

## 前置条件

确保已安装 MMB 工具：
```bash
dotnet tool update -g Materal.MergeBlock.Tools
```

## 快速开始

### 基本用法
为指定模块生成代码：
```bash
MMB GeneratorCode --ModulePath <模块路径>
```

### 模块路径查找

MMB 项目通常包含多个模块，模块路径遵循以下模式：
- `<项目根目录>\<项目名称>.<模块名称>`
- 示例：`E:\Project\知图\Server\ZhiTu.Main`（ZhiTu 是项目名称，Main 是模块名称）

使用 Glob 工具查找模块解决方案文件（`.slnx` 或 `.sln`）：
```bash
**/*.slnx
```
或
```bash
**/*.sln
```

## 工作流程

1. **识别修改的模块** - 确定哪个模块发生了变更
2. **运行代码生成** - 使用正确的模块路径执行 `MMB GeneratorCode`
3. **验证输出** - 检查是否有生成错误或警告

## 模块路径识别

当用户提及模块名称时，按以下优先级确定路径：

1. **用户明确提供的路径** - 直接使用用户指定的路径
2. **当前工作目录** - 如果在模块目录内，使用当前目录
3. **查找模块解决方案文件** - 使用 Glob 搜索匹配的 `.slnx` 或 `.sln` 文件
   - 搜索模式：`**/*<模块名>*.slnx` 或 `**/*<模块名>*.sln`
   - 如果有多个匹配，优先选择与用户意图最相关的

### 示例
- 用户说 "为 Main 模块生成代码" → 搜索 `**/*Main*.slnx` 或 `**/*Main*.sln`
- 用户说 "生成 Core 模块的代码" → 搜索 `**/*Core*.slnx` 或 `**/*Core*.sln`
- 当前在 `/Project/MyModule.Main` 目录 → 使用当前目录

## 使用触发条件

在以下情况使用此技能：
- 用户修改了实体、DTO 或配置
- 用户提及"生成代码"、"运行生成器"等类似表述
- 代码修改后准备构建项目之前
- 用户要求同步生成的代码
