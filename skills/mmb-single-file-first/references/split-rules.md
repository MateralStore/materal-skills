# 拆分阈值与边界

## 目标

在不牺牲可维护性的前提下，优先让 MMB 自定义代码保持“单文件可读”。

## 建议阈值

1. 总行数阈值：`300` 行（同一类的自定义文件总和，空行不计）。
2. 方法数阈值：`8` 个（同一类 public/protected/internal/private 方法总和）。
3. 拆分文件阈值：`3` 个（不含主文件）。

同时满足以上阈值时，判定为“简单类”，应合并到主文件。

## 直接改名场景

出现以下场景时直接改名，无需讨论：

1. 仅有一个碎片文件（如 `UserController.Crud.cs`）。
2. 主文件不存在（`UserController.cs` 不存在）。

处理方式：将该文件改名为主文件名。

## 保留拆分场景

满足任一条件即可保留拆分：

1. 总行数超过 `300`。
2. 方法数超过 `8`。
3. 同类职责明显独立且长期演进频率不同（如 `Auth` 与 `Crud`）。

## 禁止项

1. 不得修改任何 `MGC` 文件。
2. 不得为了“看起来整洁”而把复杂类强行合并成超大文件。
3. 不得在没有判定依据时保留拆分。
