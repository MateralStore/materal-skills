# MMB 需求分析

本技能提供 Materal.MergeBlock(MMB) 框架项目的需求分析流程。

## 核心原则

- **逐个提问式交互**：一次只问一个问题，避免给用户造成压力
- **选项优先**：如果问题可以通过默认选项回答，使用选项式交互工具
- **动态提问**：根据用户回答动态决定下一个问题
- **完整确认**：收集完信息后汇总请用户确认

## 信息收集清单

### 1. 功能概述
- 功能描述
- 目标用户
- 解决的业务问题

### 2. 业务场景
- 典型使用场景
- 业务流程
- 前置条件

### 3. 涉及实体
- 业务实体
- 实体关系
- 核心属性

### 4. API 接口
- HTTP 端点
- 请求方式
- 参数结构
- 响应结构

### 5. 业务规则
- 数据验证
- 业务约束
- 计算逻辑

## 工作流程

需求分析按以下步骤执行：

1. **开始分析** → 告知用户将进行需求分析，说明将采用逐个提问式交互
2. **功能概述收集** → 收集功能描述、目标用户、解决的业务问题
3. **业务场景收集** → 收集典型使用场景、业务流程、前置条件
4. **实体信息收集** → 收集业务实体、实体关系、核心属性
5. **API 接口收集** → 收集 HTTP 端点、请求方式、参数结构、响应结构
6. **业务规则收集** → 收集数据验证、业务约束、计算逻辑
7. **汇总确认** → 将收集到的信息结构化呈现给用户确认

**详细交互示例和模板**：见 [references/templates.md](references/templates.md)

### 条件判断

**功能涉及多个独立模块？**
- 是 → 每个模块分别进行需求分析
- 否 → 统一进行需求分析

**实体是否已存在？**
- 是 → 确认是否需要修改或扩展
- 否 → 记录需要新建的实体
